What is Time Complexity?

Rate at which time taken increases with respect to input size.

  • 🚫 NOT actual time in seconds
  • ✅ It's the growth pattern of operations
  • ✅ We count operations relative to N
  • 🎯 Focus on WORST CASE (Big O)
"If input doubles, how much MORE work does your code do?"
📝

🔥

Rules to Calculate

Rule 1: Drop Constants

for(int i=0; i < 5*n; i++) { }
// O(5N) → just O(N)
Constants vanish at scale 🪄

Rule 2: Drop Lower Order Terms

// O(N² + N + 1) → O(N²)
// N² dominates when N is huge

Rule 3: Different Inputs = Different Variables

for(i=0; i<n; i++)
  for(j=0; j<m; j++) { }
// This is O(N × M), NOT O(N²)

less commonly asked

// O(1) space — just a variable
int sum = 0;
for(int i=0; i<n; i++) sum += arr[i];

// O(N) space — new array
int temp[n];

// O(N) space — recursion stack
void print(int n) {
    if(n==0) return;
    print(n-1); // N calls in stack
}

Common Mistakes

❌ Saying O(2N) instead of O(N) — drop the constant!
❌ Confusing O(N²) with O(N×M) — different inputs!
❌ Ignoring recursion stack space — each call = memory!
❌ Thinking sorting is O(N) — comparison sort min is O(N log N)

⭐ The Golden Trick

Read the constraints FIRST → use the 10⁸ rule → know your max allowed complexity BEFORE coding.
