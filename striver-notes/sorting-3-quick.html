Quick Sort

Partition & Conquer — In-Place ⚡
O(1) SpaceN log N avgNot StableDivide & Conquer
🧠

Core Idea

  1. Pick a pivot (first/last/random)
  2. Place pivot at its correct position
  3. Smaller left, larger right
  4. Recursively sort left & right
Every element reaches correct spot → sorted!

Partition + quickSort

int partition(vector<int>& a, int low, int high) {
    int pivot = a[low], i = low, j = high;
    while(i < j) {
        while(a[i] <= pivot && i < high) i++;
        while(a[j] > pivot && j > low) j--;
        if(i < j) swap(a[i], a[j]);
    }
    swap(a[low], a[j]);
    return j;
}
void quickSort(vector<int>& a, int low, int high) {
    if(low < high) {
        int p = partition(a, low, high);
        quickSort(a, low, p-1);
        quickSort(a, p+1, high);
    }
}
🔍

Partition Logic

• i → first element > pivot (from left)
• j → first element < pivot (from right)
• i < j → swap
• crossed → swap pivot with j
• return j = partition index

Common Mistakes

❌Missing i<high / j>low guards
❌Swap pivot with i instead of j
❌Saying quick sort is stable (NOT)

FAANG Interview Questions

Quick Select (Kth smallest) Google Amazon 
int quickSelect(a, low, high, k) {
    int p = partition(a, low, high);
    if(p == k-1) return a[p];
    if(p > k-1)
        return quickSelect(a, low, p-1, k);
    return quickSelect(a, p+1, high, k);
}
// Avg O(N) — only one side!
Worst case fix? Theory

Random pivot / median-of-three / introsort (switch to heap sort if depth > 2logN).

Quick vs Merge? Theory

Quick: in-place, faster avg. Merge: stable, guaranteed N log N. std::sort uses introsort.

Stable? Theory

No — partition swaps non-adjacent elements. Use merge sort for stability.

⭐ Remember

Pick pivot, partition (smaller left, larger right), recurse both sides. O(N log N) avg, O(1) space, in-place. Random pivot avoids O(N²). Quick Select finds Kth in O(N).
