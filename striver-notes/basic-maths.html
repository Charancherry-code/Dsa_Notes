Digit Extraction — The Core

% 10 → get last digit  |  / 10 → remove last digit

int n = 7789;
while(n > 0) {
    int lastDigit = n % 10;  // 9, 8, 7, 7
    n = n / 10;              // 778, 77, 7, 0
}
7789→ %10 = 9→ /10 = 778→ %10 = 8→ ...till 0
TC: O(log₁₀ N) Reverse order!
This one concept solves 5 problems below 👇

Count Digits

int countDigits(int n) {
    int count = 0;
    while(n > 0) {
        count++;
        n /= 10;
    }
    return count;
}
// One-liner: (int)log10(n) + 1
TC: O(log₁₀ N)
2️⃣

Reverse a Number

Formula: rev = rev × 10 + lastDigit

int reverse(int n) {
    int rev = 0;
    while(n > 0) {
        rev = rev * 10 + n % 10;
        n /= 10;
    }
    return rev;
}
// 7789 → 9877
TC: O(log₁₀ N)
3️⃣

Check Palindrome

reverse == original? (121 ✅, 123 ❌)

bool isPalindrome(int n) {
    int dup = n, rev = 0;
    while(n > 0) {
        rev = rev*10 + n%10;
        n /= 10;
    }
    return dup == rev;
}
⚠️Store copy of N! It becomes 0 after loop.
4️⃣

Armstrong Number

sum of digit³ == number (371 → 27+343+1 = 371 ✅)

bool isArmstrong(int n) {
    int dup = n, sum = 0;
    while(n > 0) {
        int d = n % 10;
        sum += d * d * d;
        n /= 10;
    }
    return sum == dup;
}
TC: O(log₁₀ N)
5️⃣

Print All Divisors

Brute O(N) 
for(int i=1; i<=n; i++)
    if(n%i == 0) cout << i;
Optimal O(√N) 
for(int i=1; i*i<=n; i++) {
    if(n%i == 0) {
        list.push_back(i);
        if(n/i != i)
            list.push_back(n/i);
    }
}
sort(list.begin(), list.end());
💡 Key insight: If i is a factor → n/i is also a factor. Both found in one check. Pairs exist within √N.
6️⃣

Check Prime

✅ "Exactly 2 factors"
❌ NOT "divisible by 1 & itself" (makes 1 prime) 

bool isPrime(int n) {
    if(n <= 1) return false;
    int cnt = 0;
    for(int i=1; i*i<=n; i++) {
        if(n%i == 0) {
            cnt++;
            if(n/i != i) cnt++;
        }
    }
    return cnt == 2;
}
TC: O(√N)
7️⃣

GCD — Euclidean Algorithm

Greatest Common Divisor — largest number dividing both.

Brute O(min(a,b)) 
int gcd = 1;
for(int i=1; i<=min(a,b); i++)
    if(a%i==0 && b%i==0)
        gcd = i;
Euclidean O(log φ) 
int gcd(int a, int b) {
    while(a > 0 && b > 0) {
        if(a > b) a = a % b;
        else      b = b % a;
    }
    return a == 0 ? b : a;
}
Example 
GCD(52, 10)
→ 52%10 = 2 → GCD(2,10)
→ 10%2  = 0 → GCD(2,0)
→ one is 0 → GCD = 2

GCD(20, 15)
→ 20%15 = 5 → GCD(5,15)
→ 15%5  = 0 → GCD(5,0)
→ GCD = 5
⚡ Euclidean: bigger % smaller → repeat till one is 0 → other is GCD. TC: O(log_φ(min(a,b)))

Patterns — When to Use

Problem says...Apply
"extract/manipulate digits"%10 and /10 loop
"reverse a number"rev = rev*10 + last
"reads same forwards/backwards"reverse → compare with copy
"find all factors"loop till √N, pair = n/i
"check if prime"count factors till √N == 2
"GCD / HCF"Euclidean: a%b till 0
"LCM"(a × b) / GCD(a,b)
"trailing zeros in n!"count 5's: n/5 + n/25 + ...
❌

Common Mistakes

❌Not storing copy of N (becomes 0 after loop)
❌Overflow in reverse — use long long
❌Wrong prime def (1 is NOT prime)
❌O(N) for factors — use √N always
❌Subtraction Euclidean (slow) — use % modulo
❌-7%10 = -7 in C++ — handle negatives!

int countPrimes(int n) {
    vector<bool> s(n, true);
    int cnt=0;
    for(int i=2; i<n; i++) {
        if(s[i]) { cnt++;
            for(long j=(long)i*i; j<n; j+=i)
                s[j]=false;
        }
    } return cnt;
}
