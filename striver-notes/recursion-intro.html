What is Recursion?

A function that calls itself until a base condition is met.

void f() {
    // 1. Base case — STOP
    // 2. Do some work
    // 3. Call itself with smaller input
    f();
}
3 things to identify:
1️⃣ Base case — when to STOP
2️⃣ Work — what happens each call
3️⃣ Recursive call — function calling itself with changed input
🛑

Base Case

The condition where you STOP calling further.

void f(int count) {
    if(count == 3) return; // 🛑 STOP!
    cout << count;
    f(count + 1);
}
// f(0) → prints: 0 1 2
❌No base case = infinite recursion = CRASH
📚

Stack Space (Call Stack)

Every unfinished function call waits in the stack until the one it called returns.

┌───────────────┐
│   f(3)        │ ← hits base case → RETURNS ↩️
├───────────────┤
│   f(2)        │ ← waiting... now resumes → RETURNS ↩️
├───────────────┤
│   f(1)        │ ← waiting... now resumes → RETURNS ↩️
├───────────────┤
│   f(0)        │ ← waiting... now resumes → RETURNS ↩️
├───────────────┤
│   main()      │ ← done!
└───────────────┘
Each call PUSHES to stack Return = POPS from stack Space = O(depth)
💥

Stack Overflow

void f() {
    cout << 1;
    f(); // ❌ no base case!
}
// Prints 1 forever → SEGFAULT

Stack fills up → no more memory → crash

Too many calls waiting = overflow 💀
🌳

Recursion Tree

Visual shorthand for how calls branch + return.

f(0) ──→ f(1) ──→ f(2) ──→ f(3) 🛑 BASE → return
                                        ↩️ back to f(2)
                                 ↩️ back to f(1)
                          ↩️ back to f(0)
                   ↩️ back to main
💡Draw this for EVERY recursion problem. It makes the logic click instantly.
⏱️

Patterns — When Recursion?

Problem says...Think
"do something N times"f(i) → f(i+1)
"break into subproblems"classic recursion
"tree/graph traversal"DFS = recursion
"all subsets/permutations"backtracking
"try all possibilities"recursion + prune
"divide & conquer"merge sort, quick sort
"overlapping subproblems"recursion + memo = DP

Common Mistakes

❌No base case → stack overflow
❌Base case never reached (params don't change)
❌Using globals when params work better
❌Forgetting SC = O(depth) — it uses memory!
❌Tracing every call — trust the recursion

Parameterised vs Functional

Parameterised — carry answer 
void f(int i, int sum) {
    if(i < 1) { cout << sum; return; }
    f(i-1, sum + i);
}
// f(3,0) → f(2,3) → f(1,5) → f(0,6) print!

Answer built in parameter → print at base

Functional — return answer 
int f(int n) {
    if(n == 0) return 0;
    return n + f(n-1);
}
// f(3) = 3+f(2) = 3+2+f(1) = 3+2+1+0 = 6

Break into current + f(smaller) → DP uses this!

⚡Functional = what DP is built on. Master this pattern now.
🎯

Factorial (Functional)

int factorial(int n) {
    if(n == 0) return 1; // ⚠️ NOT 0!
    return n * factorial(n-1);
}
// 4! = 4×3×2×1×1 = 24
Return 0 → everything × 0 = 0. Return 1!
🔥

Backtracking Concept

Code before call = going deeper. Code after call = coming back.

Print BEFORE call (normal) 
void f(int i, int n) {
    if(i > n) return;
    cout << i;        // ← print first
    f(i+1, n);        // then go deeper
}
// f(1,3) → 1 2 3
Print AFTER call (backtrack) 
void f(int i, int n) {
    if(i < 1) return;
    f(i-1, n);        // go deep first
    cout << i;        // ← print on way BACK
}
// f(3,3) → 1 2 3 (started from 3!)
⚡ Rule: Before call = going down. After call = coming back up. This is backtracking in its simplest form.
📝

⭐ Golden Rule

Trust the recursion. If f(smaller) gives the correct answer, just build on top of it. Don't trace every single call — think in terms of "what does this ONE call do?"
